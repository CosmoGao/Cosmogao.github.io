搭建基于 Hexo 和 Git Pages 的个人技术博客

发表于 2016-09-18 更新于 2017-10-12 分类于 技术文档

导言

搭建一个个人博客需要什么？

一般来说，建立一个网站所必需的要素有以下几个内容：

  • 主机（Git Pages）
  • 域名（Freenom）
  • 内容（Hexo）

    Git Pages 是什么？

GitHub Pages is a static site hosting service.
GitHub Pages is designed to host your personal, organization, or project pages directly from a GitHub repository
GitHub Pages 是一个静态网站托管服务。
GitHub Pages 可以从一个 GitHub 仓库来建立个人、组织或者项目页面。

通俗地讲，Git Pages 是 GitHub 提供的一项静态页面托管服务，服务依托于 GitHub 仓库，用户可以利用这项功能，简单、快速的建立个人、组织或项目的静态页面。我们这里就选择 GitHub Pages 作为个人网站的主机，GitHub Pages 的主机服务，提供了一个 username.github.io 的二级域名，采用了 SSL 加密，也可以绑定自己的顶级域名，这里推荐 Freenom，免费提供 .tk/.ml 等后缀的顶级域名。

Hexo

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

解决了主机和域名的问题，接下来就需要解决内容的问题，由于 Git Pages 只能支持静态界面，也不支持数据库，所以这里没有办法搭建如 WordPress 之类的博客，Git Pages 本身提供了 Automatic Page Generator 工具，可以生成简单的基础页面，你也可以将自己的静态页面提交到仓库，进行展示。

不过 Hexo 和 Jekyll 为我们提供了一个更为简单的方法，分别基于 Node.js 和 Ruby 开发，可以在本地快速、简单的生成静态网页，我们这里选择 Hexo 作为我们的生成工具。

建立 GitHub Pages 仓库

如果你之前没有注册过 GitHub，请先注册一个 GitHub 账号，你的 GitHub 账号就是以后 Git Pages 二级域名的前缀。

登录 GitHub 之后，新建（New）一个仓库（Repository），注意仓库的名字必须是 username.github.io，其中的 username 是你的 GitHub 用户名。

之后，你可以在 Repo 中新建一个 index.html 文件，测试一下是否能够访问 username.github.io，如果能够访问，那么你已经完成了 Git Pages 的建立工作。
